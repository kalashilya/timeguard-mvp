# Prompt 02 — MVP scope

Задача: определить минимальный объём TimeGuard Planner для финальной демонстрации.

Ограничения:

- проект должен работать как статический сайт;
- один основной end-to-end сценарий;
- данные должны сохраняться в браузере;
- должна быть понятная логика приложения;
- нужны профиль, роли, лимит Free и тарифный шаг;
- не подключать сложный backend до готовности основного MVP.

Итоговый scope:

- лендинг;
- профиль;
- форма задач;
- проверка пересечений;
- сортировка;
- кабинет;
- тарифы;
- учебная admin-сводка;
- документация для защиты.
