# Демонстрационный сценарий TimeGuard Planner

## Цель

Показать полный путь пользователя: от проблемы хаотичного планирования до рабочего планировщика с проверкой конфликтов, экспортом, тарифным сценарием и Supabase-синхронизацией.

## Быстрый старт

Начать защиту лучше со страницы:

`demo-center.html`

На ней собраны быстрые ссылки, сценарий показа и проверка состояния MVP.

## Сценарий показа

1. Открыть `demo-center.html`.
2. Показать, что проект опубликован и готов к демонстрации.
3. Открыть `app.html`.
4. Показать проблему: обычный список дел не проверяет пересечения.
5. Создать или показать профиль пользователя.
6. Нажать «Заполнить примером» или вручную добавить задачи.
7. Показать сортировку задач по времени.
8. Попробовать добавить задачу с пересечением.
9. Показать предупреждение о конфликте времени.
10. Отметить одну задачу выполненной.
11. Показать прогресс выполнения.
12. Показать фильтры по категории и приоритету.
13. Показать недельный обзор.
14. Экспортировать план в TXT или JSON.
15. Открыть кабинет.
16. Нажать синхронизацию с Supabase.
17. Открыть Cloud-страницу и показать загрузку задач из базы.
18. Открыть тарифы и показать Free-лимит.
19. Открыть README, test-plan и SQL-схему.

## Что доказывает проект

- есть понятная проблема и пользователь;
- есть рабочий пользовательский сценарий;
- есть данные и хранение;
- есть Supabase Auth и база;
- есть RLS-политики;
- есть синхронизация задач;
- есть чтение задач обратно из базы;
- есть логика проверки конфликтов;
- есть прогресс, фильтры, недельный обзор и экспорт;
- есть Free-лимит и тарифный шаг;
- есть GitHub-документация.

## Короткий текст для защиты

TimeGuard Planner — это учебный MVP умного планировщика дня. Его задача — не просто хранить список дел, а проверять, насколько реалистично расписание. Пользователь добавляет задачи, а сервис заранее предупреждает о пересечениях по времени. В проекте реализованы профиль, кабинет, проверка конфликтов, прогресс выполнения, фильтры, недельный обзор, экспорт плана, тарифный сценарий, Supabase Auth, SQL-схема и синхронизация задач с базой данных.
