# Демонстрационный сценарий TimeGuard Planner

## Цель

Показать полный путь пользователя: от проблемы хаотичного планирования до сохранённого плана дня, проверки конфликтов и учебного тарифного шага.

## Сценарий показа

1. Открыть `app.html`.
2. Показать первый экран и проблему: обычный список дел не проверяет пересечения.
3. Создать учебный профиль пользователя.
4. Нажать «Заполнить примером» или вручную добавить задачи:
   - 10:00–11:30 — разобрать учебный проект;
   - 12:00–13:00 — рабочая встреча;
   - 18:30–19:30 — тренировка.
5. Показать, что задачи сортируются по времени.
6. Попробовать добавить задачу 10:30–11:00.
7. Показать предупреждение о конфликте времени.
8. Открыть кабинет и показать сохранённые планы по датам.
9. Достичь Free-лимита или открыть тарифы.
10. Показать учебную активацию тарифа.
11. Вернуться в кабинет и показать обновлённый план.
12. Сохранить профиль с ролью `admin` и открыть admin-страницу.

## Что доказывает проект

- есть понятная проблема и пользователь;
- есть рабочий пользовательский сценарий;
- есть данные и хранение;
- есть роли `guest`, `user`, `admin`;
- есть логика приложения;
- есть проверка конфликтов;
- есть Free-лимит и тарифный шаг;
- есть GitHub-документация.

## Короткий текст для защиты

TimeGuard Planner — это учебный MVP умного планировщика дня. Его задача — не просто хранить список дел, а проверять, насколько реалистично расписание. Пользователь добавляет задачи, а сервис заранее предупреждает о пересечениях по времени. В проекте реализованы профиль, кабинет, хранение задач, роли, лимит бесплатного плана и учебный тарифный сценарий.
