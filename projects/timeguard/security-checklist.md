# Security checklist TimeGuard Planner

## Цель

Показать, что даже учебный MVP учитывает базовые риски: ошибки ввода, повторные клики, спам формы, неверное время, защиту пользовательских данных и безопасное подключение Supabase.

## Frontend-проверки

- обязательные поля задачи;
- окончание задачи должно быть позже начала;
- запрет пересечения задач по времени;
- чекбокс «Я не робот» перед добавлением задачи;
- cooldown между отправками формы;
- Free-лимит по количеству задач и датам планирования;
- разделение ролей `guest`, `user`, `admin` для учебного интерфейса;
- admin-страница открывается только при роли `admin` в frontend demo.

## Supabase security

- используется только publishable key;
- secret/service-role key не используется во frontend;
- включён RLS для `profiles`;
- включён RLS для `tasks`;
- включён RLS для `task_events`;
- пользователь может читать только свой профиль;
- пользователь может читать только свои задачи;
- пользователь может создавать и обновлять только свои задачи;
- Cloud-страница получает данные только текущего пользователя.

## Что защищает MVP

- от ошибочного времени;
- от случайных повторных кликов;
- от добавления невыполнимого плана;
- от чтения чужих задач через Supabase;
- от случайной публикации secret key;
- от смешивания данных разных пользователей.

## Ограничения

- тарифная оплата является учебной имитацией;
- admin-доступ является frontend demo;
- production admin должен работать через backend;
- production тарифы должны проверяться на сервере;
- production rate limit лучше вынести в Edge Functions.

## Что добавить после MVP

- Edge Function для создания задач;
- server-side проверка конфликтов;
- server-side rate limit;
- payment webhooks;
- backend admin summary;
- audit logs;
- realtime security testing.

## Финальная формулировка для защиты

В текущем MVP безопасность разделена на два уровня: frontend-проверки защищают пользовательский сценарий от ошибок, а Supabase RLS защищает данные пользователя в базе. В браузере используется только publishable key, service-role key не используется.
