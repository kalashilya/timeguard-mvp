# Security checklist TimeGuard Planner

## Цель

Показать, что даже учебный MVP учитывает базовые риски: ошибки ввода, повторные клики, спам формы, неверное время и ограничение доступа к admin-сводке.

## Реализованные проверки

- обязательные поля задачи;
- окончание задачи должно быть позже начала;
- запрет пересечения задач по времени;
- чекбокс «Я не робот» перед добавлением задачи;
- cooldown между отправками формы;
- Free-лимит по количеству задач и датам планирования;
- разделение ролей `guest`, `user`, `admin`;
- admin-страница открывается только при роли `admin`.

## Что защищает MVP

- от ошибочного времени;
- от случайных повторных кликов;
- от добавления невыполнимого плана;
- от доступа к учебной admin-сводке без роли admin.

## Ограничения

В проекте нет настоящего backend, поэтому защита является учебной frontend-реализацией. Для production-версии нужны серверная база, авторизация, RLS и server-side rate limit.

## Что добавить после MVP

- Supabase Auth;
- PostgreSQL-таблицы для задач и профилей;
- RLS для разделения данных;
- серверный rate limit;
- настоящая проверка тарифного статуса на сервере.
