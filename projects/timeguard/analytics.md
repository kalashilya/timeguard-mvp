# Аналитика TimeGuard Planner

## Зачем нужна аналитика

Аналитика нужна, чтобы понять, проходит ли пользователь основной сценарий: видит ли ценность, создаёт ли профиль, добавляет ли задачи, сталкивается ли с предупреждением о конфликте, использует ли экспорт и доходит ли до Supabase-синхронизации.

## Рекомендуемые инструменты

Для учебного MVP:

- ручной тест-план;
- вопросы после пилотного теста;
- Supabase `task_events` для будущих событий;
- Microsoft Clarity или аналог для поведения на странице.

## Что отслеживать

- открытие Demo Center;
- клик по «Подготовить демо-данные»;
- клик по CTA «Начать планирование»;
- создание профиля;
- регистрация/логин через Supabase;
- добавление задачи;
- появление конфликта времени;
- отметка задачи как выполненной;
- использование фильтров;
- открытие недельного обзора;
- экспорт TXT;
- экспорт JSON;
- открытие кабинета;
- синхронизация с Supabase;
- открытие Cloud-страницы;
- переход на тарифы;
- подтверждение учебного тарифа;
- открытие admin-страницы.

## Что считать успехом MVP

- пользователь понял ценность за 30-60 секунд;
- пользователь создал или увидел профиль;
- пользователь добавил минимум две задачи;
- пользователь увидел предупреждение о конфликте;
- пользователь отметил хотя бы одну задачу выполненной;
- пользователь понял прогресс выполнения;
- пользователь открыл кабинет;
- пользователь синхронизировал задачи с Supabase;
- пользователь увидел задачи на Cloud-странице;
- пользователь понял ограничение Free-плана.

## Метрики

| Метрика | Цель |
|---|---|
| Время составления плана | 3-5 минут |
| Понимание ценности | 8/10 и выше |
| Успешное добавление задач | 80%+ тестовых пользователей |
| Понимание конфликта | 80%+ тестовых пользователей |
| Использование экспорта | 30%+ тестовых пользователей |
| Успешная синхронизация | 90%+ тестовых запусков |
| Готовность пользоваться | 60%+ пользователей |

## Что улучшать по данным

- если мало кликов по CTA — усилить первый экран;
- если не добавляют задачи — упростить форму;
- если не понимают конфликт — сделать предупреждение заметнее;
- если не открывают кабинет — лучше объяснить его пользу;
- если не синхронизируют Supabase — сделать cloud-сценарий понятнее;
- если не используют экспорт — добавить пояснение, зачем он нужен.
