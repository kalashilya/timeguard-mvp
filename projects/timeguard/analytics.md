# Аналитика TimeGuard Planner

## Зачем нужна аналитика

Аналитика нужна, чтобы понять, проходит ли пользователь основной сценарий: видит ли ценность, создаёт ли профиль, добавляет ли задачи и сталкивается ли с предупреждением о конфликте.

## Рекомендуемый инструмент

Microsoft Clarity.

## Что отслеживать

- клик по CTA «Начать планирование»;
- создание профиля;
- добавление задачи;
- появление конфликта времени;
- открытие кабинета;
- переход на тарифы;
- подтверждение учебного тарифа;
- открытие admin-страницы.

## Что считать успехом MVP

- пользователь создал профиль;
- добавил минимум две задачи;
- увидел предупреждение о конфликте;
- открыл кабинет;
- понял ограничение Free-плана;
- дошёл до тарифного шага.

## Что улучшать по данным

- если мало кликов по CTA — усилить первый экран;
- если не добавляют задачи — упростить форму;
- если не понимают конфликт — сделать предупреждение заметнее;
- если не открывают кабинет — лучше объяснить его пользу.
