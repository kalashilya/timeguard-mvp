# User flow TimeGuard Planner

## Основной сценарий

1. Пользователь открывает TimeGuard Planner.
2. Видит ценность: план дня без конфликтов времени.
3. Создаёт учебный профиль.
4. Добавляет первую задачу с датой, временем начала и окончания.
5. Добавляет ещё одну задачу.
6. Система сортирует задачи по времени.
7. Пользователь пытается добавить задачу, которая пересекается с уже существующей.
8. Система показывает предупреждение и не сохраняет конфликтную задачу.
9. Пользователь открывает кабинет и видит планы по датам.
10. При достижении Free-лимита пользователь переходит на тарифный экран.

## Ключевой критерий

Пользователь должен за 1–2 минуты получить первый полезный результат: сохранённый план дня и понятное предупреждение о конфликте времени.

## Альтернативный сценарий

1. Пользователь открывает сайт без профиля.
2. Добавляет задачи в режиме гостя.
3. Сохраняет профиль позже.
4. Возвращается к планам через кабинет.

## Ошибочные сценарии

- пользователь не заполнил обязательные поля;
- окончание задачи раньше начала;
- задача пересекается с другой задачей;
- пользователь слишком быстро повторно нажимает кнопку;
- пользователь достиг Free-лимита.
