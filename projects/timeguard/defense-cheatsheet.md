# Шпаргалка для защиты TimeGuard Planner

## Главная мысль

TimeGuard Planner — это учебный MVP умного планировщика дня. Он помогает пользователю не просто хранить задачи, а собрать реалистичное расписание без пересечений по времени.

## С чего начинать показ

Открыть Demo Center:

`projects/timeguard/demo-center.html`

На этой странице есть подготовка демо-данных, проверка состояния MVP и быстрые ссылки на все ключевые разделы.

## Проблема

Обычный todo-list показывает список задач, но не проверяет, можно ли реально выполнить план. Пользователь может поставить две задачи на одно время и увидеть проблему слишком поздно.

## Пользователь

Студент, молодой специалист или менеджер, которому нужно совмещать учёбу, работу, встречи и личные дела.

## Ценность

Пользователь быстрее собирает реалистичный план дня, заранее видит конфликты, контролирует выполнение задач и может экспортировать расписание.

## Что реализовано

- лендинг;
- профиль пользователя;
- register/login/verify flow;
- планировщик задач;
- проверка конфликтов по времени;
- сортировка задач;
- прогресс выполнения;
- фильтры по категории и приоритету;
- недельный обзор;
- экспорт TXT и JSON;
- кабинет;
- Supabase Auth;
- синхронизация задач с Supabase;
- Cloud-страница для чтения задач из базы;
- Free-лимит и тарифный сценарий;
- admin-страница;
- README, test plan, SQL-схема и сценарий защиты.

## Основной сценарий защиты

1. Открыть Demo Center.
2. Нажать «Подготовить демо-данные».
3. Нажать «Проверить статус MVP».
4. Открыть Pitch и коротко объяснить продукт.
5. Открыть планировщик.
6. Показать задачи, прогресс, фильтры и недельный обзор.
7. Попробовать добавить задачу с пересечением.
8. Показать предупреждение.
9. Экспортировать план.
10. Открыть кабинет.
11. Синхронизировать с Supabase.
12. Открыть Cloud-страницу и показать задачи из базы.
13. Показать тарифы и Free-лимит.
14. Показать README и SQL-схему.

## Данные

- профиль пользователя;
- роль;
- тариф;
- дата задачи;
- название;
- время начала;
- время окончания;
- приоритет;
- категория;
- комментарий;
- статус выполнения;
- local_id для синхронизации с Supabase.

## Хранение

В MVP используются два уровня хранения:

- localStorage — быстрый frontend-сценарий;
- Supabase — реальная база данных, auth-flow и синхронизация задач.

## Безопасность

- обязательные поля;
- проверка времени начала и окончания;
- проверка пересечений;
- чекбокс «Я не робот»;
- cooldown формы;
- RLS-политики Supabase;
- пользователь видит только свои задачи;
- service role key не используется во frontend.

## Речь на 40 секунд

Я сделал TimeGuard Planner — учебный MVP умного планировщика дня. Проблема в том, что обычные списки дел не проверяют, реально ли выполнить расписание. Пользователь может поставить две задачи на одно время и заметить конфликт слишком поздно. TimeGuard решает это: пользователь добавляет задачи с датой и временем, а система заранее проверяет пересечения, сортирует задачи и не даёт сохранить невыполнимый план. В проекте есть профиль, кабинет, прогресс выполнения, фильтры, недельный обзор, экспорт плана, Free-лимит, тарифный сценарий, Supabase Auth, SQL-база, RLS и cloud-страница, которая читает задачи обратно из базы.
