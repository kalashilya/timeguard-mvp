# Presentation outline

## Слайд 1. TimeGuard Planner

**TimeGuard Planner** — учебный MVP умного планировщика дня без конфликтов времени.

Коротко: сервис помогает собрать реалистичный план дня и заранее увидеть, если задачи пересекаются по времени.

## Слайд 2. Проблема

Обычный todo-list хранит задачи, но не проверяет, реально ли выполнить расписание.

Пользователь может поставить две задачи на одно время и заметить это слишком поздно.

## Слайд 3. Пользователь и ценность

**Пользователь:** студент, молодой специалист или менеджер.

**Ценность:** пользователь быстрее собирает реалистичный план дня, видит конфликты и получает больше контроля.

## Слайд 4. Основной сценарий MVP

- открыть Demo Center;
- подготовить демо-данные;
- открыть Planner;
- добавить задачу;
- увидеть сортировку;
- проверить конфликт;
- отметить задачу выполненной;
- посмотреть прогресс и фильтры.

## Слайд 5. Product features

- задачи по датам;
- время начала и окончания;
- приоритет и категория;
- проверка пересечений;
- прогресс выполнения;
- фильтры;
- недельный обзор;
- экспорт TXT/JSON.

## Слайд 6. Data and Supabase

- localStorage используется для стабильного frontend-сценария;
- Supabase используется для Auth, profiles, tasks, RLS и cloud-чтения;
- кабинет синхронизирует задачи;
- Cloud-страница читает задачи обратно из базы.

## Слайд 7. Безопасность

- обязательные поля;
- проверка времени;
- проверка конфликтов;
- cooldown формы;
- RLS-политики;
- пользователь видит только свои данные;
- service-role key не используется во frontend.

## Слайд 8. Монетизация

- Free-план ограничивает количество задач и горизонт планирования;
- Plus и Team показывают возможную коммерческую модель;
- оплата является учебной имитацией;
- в production нужны backend webhooks.

## Слайд 9. Метрики и проверка

- время составления плана;
- количество найденных конфликтов;
- доля выполненных задач;
- использование фильтров;
- использование экспорта;
- готовность пользоваться;
- готовность платить.

## Слайд 10. Что дальше

- cloud-first planner;
- realtime sync;
- календарные интеграции;
- push-уведомления;
- Edge Functions;
- production admin;
- мобильная версия.

## Финальная фраза

TimeGuard — учебный MVP, но в нём уже есть полный пользовательский путь: проблема, интерфейс, логика, данные, Supabase, безопасность, монетизация и документация для дальнейшего развития.
