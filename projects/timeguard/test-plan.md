# Test plan TimeGuard Planner

| Проверка | Ожидаемый результат |
|---|---|
| Открыть сайт | Главная страница работает |
| Нажать «Открыть приложение» | Открывается `app.html` |
| Сохранить профиль | Профиль отображается в блоке статуса |
| Добавить задачу | Задача появляется в плане дня |
| Обновить страницу | Данные сохраняются |
| Добавить задачу с пересечением | Появляется предупреждение |
| Поставить окончание раньше начала | Появляется ошибка |
| Не поставить «Я не робот» | Задача не сохраняется |
| Быстро нажать кнопку несколько раз | Срабатывает cooldown |
| Отметить задачу выполненной | Прогресс выполнения меняется |
| Использовать фильтр приоритета | Список задач фильтруется |
| Использовать фильтр категории | Список задач фильтруется |
| Проверить недельный обзор | Видны 7 дней и количество задач |
| Нажать экспорт TXT | Скачивается текстовый план |
| Нажать экспорт JSON | Скачивается JSON-файл |
| Достичь Free-лимита | Появляется тарифное окно |
| Активировать тариф | План пользователя меняется |
| Зарегистрироваться через Supabase | Профиль создаётся |
| Синхронизировать задачи | Задачи появляются в Supabase |
| Открыть Cloud-страницу | Задачи загружаются обратно из Supabase |
| Открыть admin без роли | Показывается ограничение доступа |
| Выбрать роль admin | Admin-сводка открывается |

## Финальная проверка перед защитой

1. Открыть сайт в новом браузере.
2. Создать профиль.
3. Добавить 2–3 задачи.
4. Добавить конфликт.
5. Отметить задачу выполненной.
6. Проверить прогресс.
7. Проверить фильтры.
8. Проверить недельный обзор.
9. Проверить экспорт.
10. Открыть кабинет.
11. Синхронизировать с Supabase.
12. Открыть Cloud-страницу и показать загрузку из базы.
13. Проверить GitHub README и документы.
